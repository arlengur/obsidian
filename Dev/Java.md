# Formating

String.format в Java - это статический метод класса String

Первым параметром передается строка-шаблон, в которой, на местах, в которые мы хотим подставить значения, стоят специальные символы: %s, %d и тд. 

После строки-шаблона передаются параметры, значения которых и будут подставлены на место символов %s, %d.

| Обозначение | Тип                               |
| ----------- | --------------------------------- |
| %s          | String                            |
| %d          | целое число: int, long, …         |
| %f          | вещественное число: float, double |
| %b          | boolean                           |
| %c          | char                              |
| %t          | Date                              |
| %%          | Символ %                          |
# Data types

В Java типы данных делят на две большие группы: 
- примитивные и 
- ссылочные. 
В состав примитивных типов (или просто примитивов) входят четыре подвида и восемь типов данных:
1) целые числа
	- byte, 
	- short, 
	- int, 
	- long
2) числа с плавающей точкой 
	- float, 
	- double
3) логический (boolean);
4) символьный (char).

Ссылочные типы данных ещё называют ссылками. К ним относятся 
- все классы, 
- интерфейсы, 
- массивы, 
- а также тип данных String.

Тип byte. 
Эти переменные используют, чтобы работать с потоком данных, который получили из файла или по сети.

Тип short. 
По сравнению с byte у него увеличенный, но всё же ограниченный диапазон значений. Применяют short редко — например, когда нужно экономить память.

Тип int. 
В языке Java int — самый популярный тип целочисленных данных. При вычислениях в виртуальной машине остальные целочисленные типы (byte, short) занимают столько же памяти, сколько int.

Тип long 
Применяют, когда нужно работать с большими целочисленными значениями.
По умолчанию компилятор воспринимает целое число как int, а например 9 223 372 036 854 намного больше его максимального значения, поэтому в коде программы нужно явно указать тип long: `9223372036854L`.

Тип данных double 
Используют для работы с десятичными числами.

Тип float 
Используют как экономичный вариант хранения больших массивов данных с плавающей точкой.
Когда переменной присваивают тип float, язык Java воспринимает её как тип данных double. Чтобы этого не происходило, нужно добавлять в конце переменной символ f или F.
Не стоит использовать float, когда в вычислениях нужна точность больше пяти знаков после запятой. 

Тип boolean
Используют чтобы работать с логическими значениями.
У такой переменной может быть только два значения: false (ложь) и true (истина).

Тип данных char 
Используют, чтобы хранить в переменных любые 16-разрядные символы Unicode. Но их нужно записывать строго в одинарные кавычки ' ', и только по одному.

# Значения по умолчанию для ссылочных типов данных

В плане дефолтных значений ссылочные переменные по умолчанию имеют значение — null: это означает отсутствие ссылки или то, что ссылка ни на что не указывает.
Но если вызвать метод объекта от переменной со значением null, это приведёт к ошибке NullPointerException

```java
Cat barsik = null;
barsik.meow(); // NullPointerException, потому что у переменной barsik значение null
```

# Record

Запись (Record) в Java — это класс, предназначенный для хранения данных. Он похож на традиционный класс Java, но более легкий и обладает некоторыми уникальными функциями. Записи неизменяемы по умолчанию, что означает, что их состояние не может быть изменено после их создания. 
Также это способ создания пользовательского типа данных, состоящего из набора полей или переменных, а также методов доступа к этим полям и их изменения.

```java
record Record_Name(Fields....)
```

Создадим запись Book с двумя полями: name и price. Ключевое слово public указывает, что к этой записи можно получить доступ вне пределов пакета, в котором она определена.
```java
public record Book(String name, double price) { }
```

Чтобы использовать запись в Java, мы можем создать ее экземпляр, применяя ключевое слово new
```java
Book book = new Book( "Core Java" , 324.25);
```

После создания записи к ее полям можно получить доступ, используя имя самого поля
```java
String name = book.name(); 
double price = book.price();
```

Поля записи неявно являются final и не могут быть переназначены после создания записи, а сам класс записи неизменяем.

В дополнение к автоматически сгенерированным методам доступа мы также можем определить наши собственные методы в записи, как и в обычном классе.
```java
public record Person(String name, int age) { 
	public void sayHello() { 
		System.out.println("Hello, my name is " + name); 
	} 
}
```

Записи также имеют некоторые встроенные методы, например, toString(), equals() и hashCode().

Мы можем добавить новое поле, но это не рекомендуется. Новое поле, добавляемое в запись, должно быть статическим.
```java
public record Person(String name, int age) { 
	static String sex; 
}
```

## Конструкторы в Records
### Канонический конструктор
Компилятор Java по умолчанию предоставляет нам конструктор со всеми аргументами (конструктор со всеми полями), который присваивает свои аргументы соответствующим полям. Мы также можем добавить бизнес-логику, такую ​​как условные операторы, для проверки данных.
```java
public record Person(String name, int age) { 
	public Person(String name, int age) { 
		if (age < 18) { 
			throw new IllegalArgumentException("You are not allowed to participate in general elections"); 
		} 
	} 
}
```

### Компактный конструктор
Компактные конструкторы игнорируют все аргументы, включая круглые скобки. Назначение соответствующих полей происходит автоматически.
```java
public record Person(String name, int age) { 
	public Person { 
		if (age < 18) { 
			throw new IllegalArgumentException("You are not allowed to participate in general elections"); 
		} 
	} 
}
```

### Общие классы записей
Универсальный класс записи — это класс записи, который имеет один или несколько параметров типа.
```java
public record Pair<T, U>(T first, U second) {}
```

Универсальный класс записи Pair, принимает два параметра типа T и U. Экземпляр этой записи мы можем создать следующим образом:
```java
Pair<String, Integer>pair = new Pair<>( "Hello" , 123);
```

### Вложенный класс внутри Record

```java
public record Person(String name, int age, Contact contact){

    public static class Contact {

       private final String email;
       private final String phone;

       public Contact(String email, String phone){
           this.email = email;
           this.phone = phone;
       }

       public String getEmail(){
          return email;
       }

       public String getPhone(){
          return phone;
       }
    }
}
```

Экземпляр Person мы можем создать следующим образом:
```java
Person person = new Person("John",30, new Person.Contact("john@example.com", "123-456-7890"));
```

### Вложенный интерфейс внутри Record

```java
public record Book(String title, String author, Edition edition){
    public interface Edition{
       String getName();
   }
}
```

Экземпляр Book мы можем создать следующим образом:

```java
Book book = new Book("The Hitchhiker's Guide to the Galaxy", "Douglas Adams", new Book.Edition() {
   public String getName() {
      return "Science Fiction";
   }
});
```

Запись в Java может реализовывать интерфейсы:
```java
public record Person(String name, int age) implements Printable {
   public void print() {
      System.out.println("Name: " + name + ", Age: " + age);
   }
}
```
