Медленное выполнение SQL-запроса 

Основные причины:
- Отсутствие или неправильные индексы: База данных вынуждена читать всю таблицу (Full Table Scan) вместо использования индекса, что критично для больших таблиц.
- Устаревшая статистика: Оптимизатор запросов делает неверные предположения о данных и строит «плохой» план выполнения.
- Неэффективный план выполнения (Execution Plan): Использование полного сканирования таблицы (Table Scan) вместо индексированного поиска (Index Seek) или неправильный порядок объединения таблиц.
- Блокировки и ожидание ресурсов: Запрос долго ждет своей очереди, если к данным обращаются другие запросы, или при нехватке ресурсов (процессор, I/O).
- Отсутствие партиционирования, база данных вынуждена сканировать огромные объемы данных (Full Table Scan).
- Избыточные данные и «тяжелые» функции:
	- SELECT * загружает ненужные колонки, увеличивая нагрузку на сеть и память.
	- Использование функций (WHERE LOWER(name) = ...) делает невозможным использование обычных индексов.
	- Частое использование DISTINCT, ORDER BY или больших JOIN. 

- Как диагностировать:
	- Изучить план выполнения запроса с помощью EXPLAIN (PostgreSQL/MySQL) или SET SHOWPLAN_ALL (MS SQL), проанализировать время ожидания, проверить использование индексов.
